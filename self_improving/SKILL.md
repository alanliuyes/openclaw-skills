# Self-Improving Skill

Captures learnings, errors, and corrections to enable continuous improvement. Use when: (1) A command or operation fails unexpectedly, (2) User corrects Claude ('No, that's wrong...', 'Actually...'), (3) User requests a capability that doesn't exist, (4) An external API or tool fails, (5) Claude realizes its knowledge is outdated or incorrect, (6) A better approach is discovered for a recurring task. Also review learnings before major tasks.

## Usage

Use this skill for:
- Capturing learnings from errors
- Recording user corrections
- Tracking capability gaps
- Documenting API failures
- Recording knowledge updates
- Improving recurring tasks

## Files

- memory.md - Core learnings
- reflections.md - Reflections on mistakes
- corrections.md - User corrections
- operations.md - Operational patterns
- boundaries.md - Capability boundaries
